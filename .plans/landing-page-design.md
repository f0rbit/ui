# Landing Page Design Plan for @f0rbit/ui Documentation

## Executive Summary

This plan outlines the design for a new landing page for the @f0rbit/ui component library documentation site. The design follows the existing design system: border-focused styling, minimal aesthetics, OKLCH color tokens, and the Starlight documentation framework.

The landing page will replace the current minimal splash page with 5 rich sections that showcase the library's value proposition while maintaining visual consistency.

---

## Design System Reference

### Color Tokens
- **Backgrounds**: `--bg`, `--bg-alt`
- **Text**: `--fg`, `--fg-muted`, `--fg-subtle`, `--fg-faint`
- **Accent**: `--accent`, `--accent-fg`
- **Border**: `--border`

### Spacing Scale
- `--space-xs`: 0.25rem (4px)
- `--space-sm`: 0.5rem (8px)
- `--space-md`: 1rem (16px)
- `--space-lg`: 1.5rem (24px)
- `--space-xl`: 2.5rem (40px)

### Typography
- `--text-sm`: 0.875rem
- `--text-base`: 1rem
- `--text-lg`: 1.25rem
- `--text-xl`: 1.5rem

### Design Principles
1. **Border-focused**: Use borders for visual separation, not background colors
2. **Minimal**: Clean, uncluttered layouts with generous whitespace
3. **Subtle hover effects**: `--transition: 150ms ease`, `--hover-filter`
4. **Dark mode support**: All tokens adapt automatically

---

## Sections Selected (5)

1. **Hero Section** - Primary value proposition and CTA
2. **Feature Grid** - Key benefits/highlights
3. **Component Showcase** - Interactive preview of components
4. **Code Example** - Installation and quick start
5. **Footer** - Navigation links and resources

## New Custom Components Recommendations

### 1. `<FeatureCard>` Component
**Purpose**: Standardized feature highlight card for marketing pages

```tsx
interface FeatureCardProps {
  title: string;
  description: string;
  icon?: JSX.Element;
}

// CSS Classes: .feature-card, .feature-card-title, .feature-card-description
```

**Estimated LOC**: ~40 (component + CSS)

### 2. `<CodeBlock>` Enhancement
**Purpose**: Already partially exists in custom.css, but needs a proper component

```tsx
interface CodeBlockProps {
  language: string;
  code: string;
  showCopy?: boolean;
  filename?: string;
}
```

**Estimated LOC**: ~60 (component with copy functionality)

### 3. `<TabGroup>` Component  
**Purpose**: For the component showcase section tab switching

```tsx
interface TabGroupProps {
  tabs: Array<{ id: string; label: string; content: JSX.Element }>;
  defaultTab?: string;
}

// CSS Classes: .tab-group, .tab-list, .tab-trigger, .tab-trigger-active, .tab-panel
```

**Estimated LOC**: ~80 (component + CSS)

### 4. `<Section>` Layout Component
**Purpose**: Consistent section wrapper for landing pages

```tsx
interface SectionProps {
  title?: string;
  subtitle?: string;
  children: JSX.Element;
  bordered?: boolean;  // adds top border
}
```

**Estimated LOC**: ~30 (component + CSS)

### 5. `<LiveDemo>` Component
**Purpose**: Interactive component preview area for hero section

```tsx
interface LiveDemoProps {
  children: JSX.Element;
  className?: string;
}

// Simple wrapper with .live-demo styles
```

**Estimated LOC**: ~20

1. **Hero Section Style**: Confirm Variation C (split with live preview) vs simpler centered approach

2. **Component Showcase Interaction**: Tab-based requires JS state management - confirm this is acceptable for landing page or prefer static grid

3. **New Components Location**: Should new landing-page-specific components go in:
   - `docs/src/components/landing/` (docs-specific)
   - `packages/ui/src/components/` (library export)
   
   **Recommendation**: `docs/src/components/landing/` since these are marketing components, not library components

4. **Starlight Override**: Landing page will need to override Starlight's default splash template. Confirm this approach vs building a custom Astro page outside Starlight
